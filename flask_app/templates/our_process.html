Identifying the Problem: Too Many False SARSAT Alerts

Coast Guard Rescue Coordination Centers (RCCs) receive thousands of SARSAT distress alerts annually.

However, only 4-8% of these alerts are actual distress cases—the rest are false alerts due to accidental activations, outdated registrations, or environmental interference.

Each false alert requires operator review, data-entry ("casing") and often unnecessary asset deployment, creating resource strain and delayed response times for other actual emergencies (especially during hurricanes or other disasters).

Data Collection & Curation: 10 Years of SARSAT Alerts

We built a high-quality dataset using 12,700 SARSAT alerts collected over 10 years.

The dataset included key fields such as Beacon ID, First Detection Time, Latitude/Longitude, Beacon Type, Registration Expiry Date, and Case Outcome (Distress or Non-Distress).

Data Cleaning & Engineering:

  • Missing values were addressed through imputation and augmentation techniques.
  • Additional features (i.e., data fields/attributes) were engineered to improve predictive power.
  • A portion of the data was withheld for later testing, ensuring unbiased evaluation.

Exploratory Data Analysis (EDA): Uncovering Key Patterns

Before building a model, we analyzed patterns in SARSAT alerts to identify predictive trends:

  • Geographic Trends: Certain locations, particularly close to shore, had a higher rate of false alerts.
  • Beacon Type: Some beacon types (e.g., PLBs vs. EPIRBs) were more likely to be misused or falsely activated.
  • Environmental Conditions: Factors such as wind speed, visibility, and wave height influenced whether an alert was a true distress case.
  • Repeat Offenders: Some beacons had a history of multiple false alerts, indicating potential registration or user issues.

Machine Learning Model Development: Prioritizing Practicality

After extensive testing, we selected Random Forest Classifier as the best-performing algorithm.

The model was designed to maximize recall, meaning it prioritizes correctly identifying actual distress cases while minimizing unnecessary false alarms.

Key steps:

  • Trained on historical SARSAT data to learn the characteristics of real distress vs. false alerts.
  • Tested on a holdout dataset (data the model had never seen) to evaluate accuracy.
  • Threshold tuning to ensure the model’s risk assessment aligns with RCC operational needs.

SAR-DAT Model Output: Simple, Actionable Decision Aid

The final output is designed to be easily interpretable for RCC watchstanders.

When a SARSAT alert is received, the system provides:

  • Distress Probability Score: A numerical score indicating how likely the alert is a true distress case.
  • Top Contributing Factors: A breakdown of the most important reasons for the score (e.g., beacon type, location, environmental factors).

This human-machine pairing approach ensures that operators remain in control while benefiting from AI-enhanced decision-making.

User Interface & Deployment: Designed for RCCs

We built this as a lightweight, Python-based desktop tool—requiring no complex software installations.

It is fully compatible with standard RCC workstations, leveraging existing Microsoft tools.

Outputs include:

  • GIS-Integrated Map: Displays alert location, nearby SAR assets, and satellite ground tracks.
  • Automated Risk Report: Generates a concise summary to help watchstanders decide whether to escalate an alert.

Testing & Geographic Adaptation

The current version of the model was fine-tuned based on data from the South East U.S. and Carribean maritime regions. It is designed to be updated with additinoal data to allow customizatoin and fine-tuning for RCC specific regions, thus accounting for regional differences (e.g. climatological factors).

RCC operators provided feedback to refine usability, ensuring the tool meets real-world operational needs.

Implementation & Future Expansion

The system is already fully functional and ready for deployment.

Next steps include:

  • Scaling to additional RCCs.
  • Integrating with SAROPS and real-time weather feeds.
  • Automating additional SAR decision-support functions (e.g., AI-assisted case briefings).
